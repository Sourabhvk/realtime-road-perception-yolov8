PHASE 2 — DATASET CHOICE

Step 9: Choose dataset

Recommended option:
KITTI Object Detection Dataset

Reason:
- relevant to autonomous driving/perception
- good credibility
- contains cars, pedestrians, cyclists
- useful for NEURA/Mercedes-style positioning

Faster option:
Roboflow road-scene dataset already in YOLO format

Reason:
- saves annotation conversion time
- faster to finish
- less impressive than KITTI but still usable


Step 10: Final recommendation

Use KITTI if setup time is manageable.
Use Roboflow only if KITTI conversion becomes a time sink.


PHASE 3 — DATASET STRUCTURE

Step 11: Prepare YOLO-format dataset

Final dataset should look like this:

data/
└── road_dataset/
    ├── images/
    │   ├── train/
    │   └── val/
    ├── labels/
    │   ├── train/
    │   └── val/
    └── data.yaml


Step 12: Create data.yaml

Example:

path: data/road_dataset
train: images/train
val: images/val

names:
  0: car
  1: pedestrian
  2: cyclist


Step 13: If using KITTI

Convert KITTI labels to YOLO format.

KITTI labels contain:
- class name
- bbox_left
- bbox_top
- bbox_right
- bbox_bottom

YOLO labels require:
- class_id
- x_center_normalized
- y_center_normalized
- width_normalized
- height_normalized


Step 14: Keep only needed KITTI classes

Use:
- Car → car
- Pedestrian → pedestrian
- Cyclist → cyclist

Ignore:
- Van
- Truck
- Tram
- Misc
- DontCare

Optional:
Map Van/Truck also to car if you want more vehicle examples, but mention this clearly in README.

PHASE 12 — FPS BENCHMARK

Step 48: Create FPS benchmark script

Create:

scripts/benchmark_fps.py

Content:

import time
import cv2
from ultralytics import YOLO

MODEL_PATH = "models/yolov8n_road_best.pt"
VIDEO_PATH = "data/demo_video.mp4"

model = YOLO(MODEL_PATH)
cap = cv2.VideoCapture(VIDEO_PATH)

frame_count = 0
start_time = time.time()

while True:
    ret, frame = cap.read()
    if not ret:
        break

    model.predict(frame, conf=0.25, verbose=False)
    frame_count += 1

end_time = time.time()
cap.release()

elapsed = end_time - start_time
fps = frame_count / elapsed

print(f"Frames processed: {frame_count}")
print(f"Time elapsed: {elapsed:.2f} seconds")
print(f"FPS: {fps:.2f}")

PHASE 13 — FAILURE ANALYSIS

Step 52: Find failure examples

Look for:
- missed small pedestrians
- cyclists confused with pedestrians
- occluded cars
- far-away vehicles
- false positives on signs/poles
- crowded scenes


Step 53: Save 5–8 failure examples

Save to:

outputs/failure_cases/


Step 54: Write failure analysis

Example:

The detector performed well on clearly visible vehicles but struggled with small pedestrians and cyclists at long range.
YOLOv8s improved recall compared to YOLOv8n but reduced FPS on the local GTX 1050 Ti.
Most failure cases involved occlusion, small object size, and class similarity between pedestrians and cyclists.
