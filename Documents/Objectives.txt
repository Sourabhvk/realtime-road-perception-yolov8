OBJECTIVES

1. Build a real-time road-scene object detection and tracking pipeline.

2. Fine-tune YOLOv8 on road-scene data for key traffic/robotics-relevant classes:
   - car
   - pedestrian
   - cyclist

3. Use the local laptop for lightweight development and debugging.

4. Use RWTH HPC for full model training and validation.

5. Train at least one model:
   - YOLOv8n

6. If HPC time allows, compare two model sizes:
   - YOLOv8n
   - YOLOv8s

7. Evaluate detection performance using:
   - mAP50
   - mAP50-95
   - precision
   - recall
   - F1 score

8. Add object tracking using ByteTrack so objects keep IDs across video frames.

9. Benchmark inference speed separately:
   - local laptop FPS
   - HPC/GPU FPS if allowed

10. Analyze accuracy–latency trade-off:
   - YOLOv8n = faster, lighter
   - YOLOv8s = usually more accurate, slower

11. Produce demo outputs:
   - image inference results
   - video inference results
   - tracking video with object IDs
   - failure case examples

12. Create a clean GitHub repository with:
   - setup instructions
   - dataset preparation
   - training commands
   - evaluation results
   - FPS table
   - screenshots/demo visuals
   - failure analysis
   - limitations and next steps

13. Final CV positioning:
   Show practical computer vision/perception ability, not novelty.
